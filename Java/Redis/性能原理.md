# Redis 核心性能原理

Redis 能够实现高并发与低延迟，主要基于以下三个核心机制：

## 1. 完全基于内存
Redis 绝大部分数据存储与操作都在内存中进行。内存的读写速度远超传统的磁盘 I/O（如 MySQL 等关系型数据库），这是其具备极高性能的基础前提。

## 2. 单线程模型
Redis 的核心网络 I/O 和键值对读写是由单线程执行的。
* **优势：** 避免了多线程频繁的上下文切换开销；排除了并发修改数据时的锁竞争与死锁问题，极大简化了底层数据结构的设计。
* **注：** Redis 6.0 引入了多线程来处理网络请求的解析与数据读写，但核心的**命令执行阶段**依然保持单线程，以此保证操作的原子性。

## 3. I/O 多路复用机制
[Image of Redis IO multiplexing epoll architecture]
Redis 采用 `epoll` 等多路复用技术。它允许单线程同时高效监听和处理大量并发的 Socket 连接。只有当 Socket 真正产生读写事件时，Redis 才会进行处理，有效解决了阻塞 I/O 模型下的资源浪费问题。
