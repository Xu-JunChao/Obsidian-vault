# Redis 高级数据结构及应用场景

除基础的五大数据类型外，Redis 提供了适用于特定复杂场景的高级数据结构：

## 1. BitMap (位图)
* **原理：** 并非实际的数据类型，而是基于 String 类型，以比特位（bit，即 0 或 1）为单位来存储数据状态。
* **优势：** 极大节省内存空间。
* **适用场景：** 用户每日签到记录（1 亿用户的单日签到仅需约 12MB 内存）、活跃用户状态统计。

## 2. HyperLogLog
* **原理：** 提供基于概率的不精确去重计数方案（标准误差约 0.81%）。无论数据量多大，单个 HyperLogLog 最大占用空间仅为 12KB 左右。
* **适用场景：** 网站独立访客数（UV）等海量数据的近似基数统计。

## 3. Geo
* **原理：** 底层基于 ZSet 实现，用于存储地理位置信息（经度、纬度），并提供计算距离、范围查询等功能。
* **适用场景：** “附近的人”功能、地理围栏、外卖打车软件的距离与位置计算。
