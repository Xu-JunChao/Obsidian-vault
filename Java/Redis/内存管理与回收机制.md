# Redis 内存管理与回收机制

## 1. 过期键删除策略
Redis 采用 **“惰性删除 + 定期删除”** 相结合的策略来清理设置了过期时间的键：
* **惰性删除：** 键过期后系统暂不主动干预。仅当客户端尝试访问该键时，Redis 才会检查其过期状态，若已过期则主动删除并返回 null。此策略对 CPU 友好，但可能导致废弃内存无法及时释放。
* **定期删除：** Redis 后台周期性（默认每秒 10 次）执行定时任务，随机抽取部分设有过期时间的键进行检查，主动清理已过期的键，以弥补惰性删除的不足。

## 2. 内存淘汰策略
当 Redis 内存使用量达到配置的 `maxmemory` 阈值时，会触发淘汰机制保障系统可用性。核心策略包括：
* **`noeviction`：** 默认策略，不淘汰任何数据，当内存满时拒绝所有写入操作并报错。
* **`allkeys-lru`：** 在所有键中，淘汰最近最少使用（LRU, Least Recently Used）的键。这是作为全局缓存时最常用的淘汰策略。
* **`volatile-lru`：** 仅在设置了过期时间的键中，淘汰最近最少使用的键。
* **`allkeys-lfu` / `volatile-lfu`：** Redis 4.0 引入，基于最不经常使用（LFU, Least Frequently Used）算法进行淘汰，相较于 LRU 更关注访问频率。
