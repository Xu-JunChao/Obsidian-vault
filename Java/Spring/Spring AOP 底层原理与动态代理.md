# Spring AOP 底层原理与动态代理



## 1. AOP (Aspect-Oriented Programming) 核心概念
AOP 旨在将与核心业务逻辑无关，但却被各个业务模块共同调用的逻辑（如：日志记录、性能监控、权限校验、事务管理等）封装起来，提取为“切面（Aspect）”。
* **核心术语：**
  * **切面 (Aspect)：** 封装横切关注点的模块（通常是一个标注了 `@Aspect` 的类）。
  * **连接点 (Joinpoint)：** 程序执行过程中的某个特定位置（在 Spring AOP 中，连接点只限定为方法的执行）。
  * **切入点 (Pointcut)：** 匹配连接点的表达式。定义了“在哪些类、哪些方法”上织入横切逻辑。
  * **通知 (Advice)：** 在切面的某个特定连接点上执行的动作（包括 `@Before`, `@After`, `@Around` 等）。

## 2. 底层实现：动态代理机制
Spring AOP 的底层严格依赖于动态代理技术。它在运行时动态地将代码织入到类的指定方法、指定位置上。Spring 提供了两种代理机制，会根据目标类的特征自动切换：

* **JDK 动态代理（默认）：**
  * **触发条件：** 当目标类**实现了至少一个接口**时。
  * **原理：** 利用 Java 反射包下的 `Proxy` 类和 `InvocationHandler` 接口，在运行时动态生成一个实现了与目标类相同接口的代理类对象。
* **CGLIB 动态代理：**
  * **触发条件：** 当目标类**没有实现任何接口**，或者通过配置强制开启（如 Spring Boot 2.x 之后默认强制使用 CGLIB）。
  * **原理：** 底层利用 ASM 字节码操作框架，在运行时动态生成目标类的**子类**，并重写其中的业务方法来实现代理。因此，目标类和方法**不能被声明为 `final`**。
