数据中台要加的代码：Token 换包器
当数据中台的前端接口被调用时，它依然会收到前端传来的那个外部 Token。 在即将发起 Feign 调用之前，数据中台需要做一次“换包”动作：
```
java
@RestController
@RequestMapping("/api/data-center")
public class DataCenterController {
    // 这是你们微服务内部互相通信用的那个共享私钥
    @Value("${internal.jwt.secret}")
    private String internalSecret;
    @GetMapping("/resources/{id}/alerts")
    public Result<?> getResourceAlerts(@PathVariable String id, HttpServletRequest request) {
        
        // 1. 【验外部票】：从前端的 Request 拿到外部 Token，并解析出当前用户的身份
        //   (假设你们有个拦截器或者工具类已经帮你解开前端 Token 了)
        String currentUserId = SecurityUtils.getUserId();
        String currentTenantId = SecurityUtils.getTenantId();
        
        // 2. 【查权限】：数据中台自己去库里查一下，这个用户该有什么告警权限
        //    (如果是写死或者查库都可以)
        List<String> userAlertPerms = getPermissionsForUser(currentUserId); 
        // 假设查出来是 ["alert:read:own_tenant"]
        // 3. 【造内部票 (Token Exchange)】：
        //    数据中台用内部共享的密钥 (internalSecret)，当场为这个请求捏一个全新的 JWT！
        String internalJwt = Jwts.builder()
                .setSubject(currentUserId)
                .claim("tenantId", currentTenantId)
                .claim("permissions", userAlertPerms) // ★ 核心：把算好的权限塞进内部票 ★
                .setIssuedAt(new Date())
                .setExpiration(new Date(System.currentTimeMillis() + 60000)) // 内部票通常只要 1 分钟过期就够了
                .signWith(SignatureAlgorithm.HS256, internalSecret)
                .compact();
        // 4. 【调下游】：告诉 Feign 别用前端那个旧票了，用我刚捏的 internalJwt！
        AlertPageQuery query = new AlertPageQuery();
        query.setAlertType("RESOURCE");
        
        // 假设你的 FeignClient 支持在这个方法里动态传 Header 
        // @RequestHeader("Authorization") String token
        return alertFeignClient.getAlerts("Bearer " + internalJwt, query); 
    }
}
```
🎯 告警中心的视角
在这个架构下，告警中心完完全全不需要知道前端用户的存在，它甚至不知道前端用了哪种登录方式（账号密码？扫码？短信？）。

告警中心唯一知道的是：

请求来了。
请求带了一个 Token。
我用 internal.jwt.secret (内部共享密钥) 验签。
解密成功！里面说他是 T_APPLE 租户，权限是 [alert:read:own_tenant]。
