# WeKnora

## 项目概述

**WeKnora（维娜拉）** 是一款基于大语言模型（LLM）的文档理解与语义检索框架，专为结构复杂、内容异构的文档场景而打造。框架采用模块化架构，融合多模态预处理、语义向量索引、智能召回与大模型生成推理，构建起高效、可控的文档问答流程。核心检索流程基于 RAG（Retrieval-Augmented Generation） 机制，将上下文相关片段与语言模型结合，实现更高质量的语义回答。

## 核心定位

|   |   |
|---|---|
|维度|说明|
|产品类型|检索增强问答平台 (RAG) / 智能体框架|
|技术栈|Go (主业务服务) + Python (独立解析微服务) + Vue/TS (前端 UI)|
|部署模式|容器化微服务部署 (Docker Compose / K8s Helm)|

## 系统架构

前端层:

- 基于 Vue 3 + TypeScript 构建，通过 Nginx 反向代理，提供若干个核心视图模块：智能问答、Agent 管理、知识库管理、系统设置等。
    

业务层:

- ReAct Agent引擎（支持"思考 - 行动 - 观察"循环，可调用知识库检索、网络搜索、MCP 工具等工具）
    
- 混合检索（BM25 关键词检索、向量检索、知识图谱三路并行检索，通过 Rerank 重排序保证召回精度）
    
- 模型管理、会话管理（支持主流大模型及本地模型，可对多轮对话上下文拼接、引用溯源和持久化存储）
    
- MCP集成（支持 SSE/HTTP 传输方式，内置网络搜索）
    
- 组织管理（成员邀请、知识库/Agent 跨成员共享、租户隔离检索）
    
- 文档解析服务（独立的python服务，通过 gRPC 与主服务通信，提供 OCR、文档解析（PDF/Word等）、智能切块、VLM 多模态理解能力）

# Yuxi-Know

## 项目概述

**Yuxi-Know（语析）** 基于知识图谱和向量数据库的智能知识库系统，融合了 RAG（检索增强生成）技术与知识图谱技术，为用户提供智能问答和知识管理服务。

## 核心定位

|   |   |
|---|---|
|维度|说明|
|产品类型|检索增强问答平台 (RAG) / 智能体框架|
|技术栈|Python (主业务服务 + 独立解析微服务) + Vue 3 (前端 UI)|
|部署模式|容器化微服务部署 (Docker Compose)|

## 系统架构

前端层:

- 基于 Vue 3 + Vite 构建，通过 Nginx 反向代理，提供若干个核心视图模块：智能问答、Agent 管理、知识库管理、图谱可视化、系统设置。
    

业务层:

- LangGraph Agent 引擎（基于状态图的智能体编排，支持"感知 - 思考 - 行动"循环，可调用知识库检索、Skills 工具、MCP 服务等）
    
- 混合检索（ 向量检索、 知识图谱检索、 LightRAG 融合引擎）
    
- 会话管理（PostgreSQL 持久化，支持多轮对话上下文拼接、引用溯源、LangGraph Checkpointer 状态回溯）
    
- MCP集成（支持 SSE/Stdio 传输，内置网络搜索、代码解释器等工具）
    
- 任务队列系统（ARQ + Redis，异步处理文档解析/向量化/图谱构建，支持任务取消和进度追踪）
    
- 文档解析服务（内置 RapidOCR 轻量级 OCR，可选配 MinerU 高精度 VLM 解析和 PaddleX PP-StructureV3 版面分析，支持 PDF、Word、图片、HTML 等格式的智能解析与结构化提取）

curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg \
  && curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
    sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \

## 术语

1. LLM
    

https://aws.amazon.com/cn/what-is/large-language-model/

大型语言模型，也称为 LLM，是基于大量数据进行预训练的超大型[深度学习](https://aws.amazon.com/cn/what-is/deep-learning/)模型。底层转换器是一组[神经网络](https://aws.amazon.com/cn/what-is/neural-network/)，这些神经网络由具有自注意力功能的编码器和解码器组成。编码器和解码器从一系列文本中提取含义，并理解其中的单词和短语之间的关系。

2. RAG
    

https://aws.amazon.com/cn/what-is/retrieval-augmented-generation/

检索增强生成（RAG）是指对大语言模型输出进行优化，使其能够在生成响应之前引用训练数据来源之外的权威知识库。大语言模型（LLM）用海量数据进行训练，使用数十亿个参数为回答问题、翻译语言和完成句子等任务生成原始输出。在 LLM 本就强大的功能基础上，RAG 将其扩展为能访问特定领域或组织的内部知识库，所有这些都无需重新训练模型。这是一种经济高效地改进 LLM 输出的方法，让它在各种情境下都能保持相关性、准确性和实用性。

3. 向量数据库/向量嵌入
    

https://www.ibm.com/cn-zh/think/topics/vector-database

https://qdrant.tech/articles/what-is-a-vector-database/

https://www.pinecone.io/learn/vector-database/

向量数据库可存储、管理高维向量数据并为其编制索引。数据点以称为“向量”的数组形式存储，并根据相似性对其进行聚类。这种设计可实现低延迟查询，使其成为 AI 应用程序的理想选择。

向量嵌入是数据点的数字表示，可将各种类型的数据（包括非数学数据，如单词、音频或图像）转换为机器学习模型可处理的数字数组。

4. 基本向量搜索
    

近似近邻（ANN）搜索以记录向量嵌入排序顺序的索引文件为基础，根据接收到的搜索请求中携带的查询向量查找向量嵌入子集，将查询向量与子群中的向量进行比较，并返回最相似的结果。

https://milvus.io/docs/zh/single-vector-search.md

5. BM25
    

https://en.wikipedia.org/wiki/Okapi_BM25

在信息检索领域 ， **Okapi BM25** （ _BM_ 是_最佳匹配_的缩写）是[搜索引擎](https://en.wikipedia.org/wiki/Search_engine)用来估计文档与给定搜索查询相关性的排名函数 。它基于 Stephen E. Robertson 、 Karen Spärck Jones 等人在 20 世纪 70 年代和 80 年代开发的概率检索框架 。

6. 多路召回
    

也叫混合检索 (Hybrid Search)。Vector Search 支持混合搜索，这是一种信息检索 (IR) 中常见的架构模式，可同时结合使用语义搜索和关键字搜索（也称为基于词元的搜索）。借助混合搜索，开发者可以充分利用这两种方法中的优点，从而有效地提供更高的搜索质量。
